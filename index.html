PyroSQL driver · install

Database client for the PWire binary protocol. Two packages ship from this repo:

  • pyrosql-driver — the C-ABI shared library (libpyrosql_ffi_pwire.so) your application links against.
  • pyrosql-cli — the pyrosql REPL binary (psql-equivalent). Independent of the shared library; install whichever you need.

Typical install: sudo apt install pyrosql-driver pyrosql-cli — then pyrosql --host myserver.example.com --user me.

Which track do I choose?
modern works on every Linux from 2018 or newer (Ubuntu 20.04+, Debian 11+, RHEL 8+, Fedora, Amazon Linux 2023). Pick this if you’re on a current system.
legacy works all the way back to 2012 (RHEL 7, CentOS 7, Ubuntu 14.04+, Amazon Linux 2). Pick this only if the modern track fails with a glibc error on install.
Both tracks ship the same code with the same performance — only the minimum glibc version differs.

Debian · Ubuntu · Linux Mint · Pop!_OS · elementary OS

modern Ubuntu 20.04+ / Debian 11+

curl -fsSL https://PyroSQL.github.io/pyrosql-driver/gpg.key | sudo gpg --dearmor -o /usr/share/keyrings/pyrosql.gpg
echo "deb [signed-by=/usr/share/keyrings/pyrosql.gpg] https://PyroSQL.github.io/pyrosql-driver/apt/stable stable main" | sudo tee /etc/apt/sources.list.d/pyrosql.list
sudo apt update
sudo apt install pyrosql-driver pyrosql-cli

legacy Ubuntu 14.04 – 18.04 / Debian 8 – 10

curl -fsSL https://PyroSQL.github.io/pyrosql-driver/gpg.key | sudo gpg --dearmor -o /usr/share/keyrings/pyrosql.gpg
echo "deb [signed-by=/usr/share/keyrings/pyrosql.gpg] https://PyroSQL.github.io/pyrosql-driver/apt/stable-legacy stable main" | sudo tee /etc/apt/sources.list.d/pyrosql.list
sudo apt update
sudo apt install pyrosql-driver pyrosql-cli

Fedora · RHEL · Rocky · Alma · CentOS Stream · Amazon Linux

modern RHEL 8+ / Rocky 8+ / Alma 8+ / Fedora / Amazon Linux 2023

sudo tee /etc/yum.repos.d/pyrosql.repo <<'EOF'
[pyrosql]
name=PyroSQL
baseurl=https://PyroSQL.github.io/pyrosql-driver/yum/stable/$basearch/
enabled=1
gpgcheck=1
gpgkey=https://PyroSQL.github.io/pyrosql-driver/gpg.key
EOF
sudo dnf install pyrosql-driver pyrosql-cli

legacy RHEL 7 / CentOS 7 / Amazon Linux 2

sudo tee /etc/yum.repos.d/pyrosql.repo <<'EOF'
[pyrosql]
name=PyroSQL
baseurl=https://PyroSQL.github.io/pyrosql-driver/yum/stable-legacy/$basearch/
enabled=1
gpgcheck=1
gpgkey=https://PyroSQL.github.io/pyrosql-driver/gpg.key
EOF
sudo yum install pyrosql-driver pyrosql-cli

openSUSE · SUSE Linux Enterprise

sudo zypper addrepo --gpgcheck --refresh https://PyroSQL.github.io/pyrosql-driver/yum/stable/$basearch/ pyrosql
sudo rpm --import https://PyroSQL.github.io/pyrosql-driver/gpg.key
sudo zypper install pyrosql-driver pyrosql-cli

Alpine Linux

Alpine uses musl libc, so it gets its own build + its own RSA signing scheme (separate from the apt/yum GPG key). One track only — the stable repo is built against Alpine 3.19 (musl 1.2.4) and runs on any Alpine 3.17+.

sudo wget -O /etc/apk/keys/info-pyrosql-com.rsa.pub https://PyroSQL.github.io/pyrosql-driver/apk/info-pyrosql-com.rsa.pub
echo "https://PyroSQL.github.io/pyrosql-driver/apk/stable" | sudo tee -a /etc/apk/repositories
sudo apk update
sudo apk add pyrosql-driver pyrosql-cli

Verify the installation

# Driver: the shared library must be loadable
ldconfig -p | grep pyrosql
# -> libpyrosql_ffi_pwire.so (libc6,x86-64) => /usr/lib/x86_64-linux-gnu/libpyrosql_ffi_pwire.so

# CLI: the pyrosql REPL binary
which pyrosql && pyrosql --version
# -> /usr/bin/pyrosql
# -> pyrosql 1.3.x

# Check the package versions
apt show pyrosql-driver pyrosql-cli 2>/dev/null   || rpm -qi pyrosql-driver pyrosql-cli 2>/dev/null   || apk info -v pyrosql-driver pyrosql-cli

Troubleshooting

“GLIBC_2.XX not found” on install

Your system’s glibc is older than the modern track requires. Switch to the legacy repo (URL above) and reinstall.

“The following signatures couldn’t be verified”

The signing key wasn’t imported. Re-run the curl … gpg.key step above.

“Unable to locate package pyrosql-driver”

apt update didn’t see the new source. Check that /etc/apt/sources.list.d/pyrosql.list exists and re-run sudo apt update.

Uninstall

# Debian / Ubuntu
sudo apt remove pyrosql-driver pyrosql-cli
sudo rm /etc/apt/sources.list.d/pyrosql.list /usr/share/keyrings/pyrosql.gpg

# RHEL / Fedora
sudo dnf remove pyrosql-driver pyrosql-cli
sudo rm /etc/yum.repos.d/pyrosql.repo

# Alpine
sudo apk del pyrosql-driver pyrosql-cli
sudo sed -i '/pyrosql-driver\/apk\/stable/d' /etc/apk/repositories
sudo rm /etc/apk/keys/info-pyrosql-com.rsa.pub
